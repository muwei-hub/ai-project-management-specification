## 📖 项目背景

我是一个**编码能力小白**，在学习AI编程的过程中踩了无数坑：

- 😱 让AI写代码，结果写出一堆语法错误
- 🤦 需求说不清楚，AI理解偏差，返工N次
- 💣 改了一个功能，其他地方莫名其妙挂了
- 📝 文档没写，代码改了三次就忘了当初为什么这么写
- 🔒 硬编码密码、裸except异常、没有测试...

后来我意识到：**不是AI不行，是我的"项目管理能力"跟不上AI的开发速度**。

于是我开始整理这套规范，结合PMI CPMAI、MLOps、ISO 42001等框架，把踩过的坑变成硬性规则，让AI协作开发变得可控、可追溯、可交付。

这份文档是我和AI协作的产物——**人类主导思考，AI辅助整理**。

## ✨ 规范亮点

| 特性 | 说明 |
|------|------|
| **门禁驱动** | 每个阶段必须通过检查才能进入下一阶段 |
| **八条硬性规则** | 没有需求文档不编码、改动先改文档再改代码... |
| **数据管理专章** | AI项目特有的数据需求、质量门禁、隐私合规 |
| **模型评估体系** | 性能指标、漂移监控、版本管理 |
| **扣子平台适配** | 针对扣子智能体/工作流的专项规范 |
| **实战踩坑案例** | 语法错误、硬编码、裸except...附正确写法 |
